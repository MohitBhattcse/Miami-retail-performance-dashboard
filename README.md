# 📊 Miami Retail Performance Dashboard

## 🚀 Project Overview

This project focuses on analyzing retail performance across multiple shops in Miami using **SQL for data preparation** and **Power BI for visualization**.

The dashboard provides actionable insights into **sales trends, customer behavior, and environmental impact (weather conditions)**, helping stakeholders make data-driven decisions.

## 🎯 Objectives

* Analyze **sales performance across shops**
* Understand **customer demographics (gender & family type)**
* Evaluate **customer spending behavior**
* Study the impact of **weather conditions on sales**
* Build a **clean, professional, and interactive dashboard**

---

## 🛠️ Tech Stack

* **SQL (MySQL)** → Data cleaning, transformation, and merging tables
* **Power BI** → Data visualization & dashboard creation
* **ODBC Connector** → Connecting MySQL with Power BI

---

## 🗄️ Data Preparation (SQL Work)

### 🔹 Data Sources

Multiple tables containing:

* Sales data
* Customer data
* Shop details
* Weather information

---

### 🔹 Key SQL Operations Performed

* **Joins** (INNER JOIN, LEFT JOIN)
  → Combined multiple tables into a single dataset

* **Data Cleaning**
  → Removed inconsistencies and handled missing values

* **Feature Engineering**
  Created new columns such as:

  * `Sales_Per_Customer`
  * `Day_Of_Week`
  * `is_weekend`
  * `is_rain`

* **Aggregation & Filtering**
  → Prepared analysis-ready dataset

---

### 🔹 Final Dataset Includes

* `sales_usd`
* `customers`
* `Sales_Per_Customer`
* `avg_temp_f`
* `precip_in`
* `is_rain`
* `pct_male`, `pct_female`
* `pct_family`, `pct_single`
* `shop_name`, `date`
